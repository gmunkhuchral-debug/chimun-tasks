### Нэг хүн олон бүлэгт (cross-branch staff) — энэ нь архитектурд гол асуудал

| Хүн | Бүлэг | Тэмдэглэл |
|---|---|---|
| Г.Мөнх-Учрал (CEO) | Бүгд | Бүх бүлгийг харна |
| Б.Дэлгэрбат | Store + Camp | Хоёр бүлгийн ҮАХ захирал давхар |
| Бямбацэрэн | Store + Camp | Худалдан авалт + camp manager |
| Н.Анужин | Нэгдсэн + Camp | Санхүүгийн захирал + camp гал тогоо |
| И.Алтансүх | M Event + (зарим Camp event) | Production tier event-д Camp-тай хамтран |

### M Event-ийн актын 5-дамжлагат урсгал — энэ нь даалгаврын app-ын СУУРЬ ОБЪЕКТ

```
Борлуулалт (акт үүсгэх) → Засвар-Агуулах ахлах (тест) → Агуулах-Логистик (хүргэх/буцаах)
   → Цэврэлгээ (цэвэрлэж шалгах) → Нярав (хаах, бүртгэл)
```

Дамжлага бүрд гарын үсэг + огноо шаардлагатай. Нэг л алхам дутуу бол нярав акт хааж болохгүй. **Энэ нь бэлэн workflow** — даалгаврын апп үүнийг template болгон ашиглах ёстой.

### Camp ↔ M Event-ийн ХАМТРАН АЖИЛЛАХ ГЭРЭЭ (CHIMUN-AGR-2026-001) — даалгаврын app-д үүнийг тусгах ёстой

- Production tier event (280K/хүн) дээр M Event 60K/хүн хуваарь авна
- 2026 онд 12-13 Production event төлөвлөсөн (≈70-80M M Event-д орох)
- M Event-ийн бэлдэлт **арга хэмжээний өмнөх шөнө дуусах ёстой**, тоног арга хэмжээний 2 цагийн өмнө бэлэн
- Хохирлын дүрэм: 30 минутын саатал = 60K-ын 20% буцаалт

**Хэлэх гэсэн зүйл:** Даалгаврын апп нь зөвхөн "to-do list" биш, **хариуцлагын урсгалын систем** байх ёстой. Acт болон Production event нь шууд template болгон тавигдана.

### KPI системд аль хэдийн жинлэгдсэн KPI байна (M_Event_KPI_v5)

Ажилтан бүрд 4 KPI (K01–K04), нийт жин 100%, бонусийн томьёо `Үндсэн × MAX(0,(KPI−80)/20) × 20%`. Жишээ нь Захиалгын ажилтны K03 нь "+40,000,000₮ борлуулалтын зорилго". **Даалгаврын апп нь KPI-тай шууд холбогдох ёстой** — даалгавар бүрийг K-кодтой холбож, KPI оноог автоматаар тооцох боломжтой.

## ② Хувилбар A-ийн нарийн дизайн

### 2.1 Дата загвар (Google Sheet-ийн tasks хүснэгт)

Одоогийн `tasks` хүснэгтэд **3 шинэ багана нэмнэ**:

```
| id | title | desc | branch* | project | assignee | co_assignees* | due | priority | status | kpi_code* | created | updated |
```

| Шинэ багана | Жишээ утга | Тайлбар |
|---|---|---|
| `branch` | `m-event` / `camp` / `store` / `shared` / `production` | Аль бүлэг хариуцагч |
| `co_assignees` | `001,005` (comma-separated) | Хэд хэдэн хүнтэй task-д (Production event-д ашигтай) |
| `kpi_code` | `K01` / `K02` / `K03` / `K04` / `null` | KPI-той холбогдсон бол ямар K-код |

**Бүлгийн утгууд:**
- `m-event` — зөвхөн M Event-ийн ажил
- `camp` — зөвхөн Camp-ийн ажил
- `store` — зөвхөн Store-ийн ажил
- `shared` — Нэгдсэн алба (Тэмүүлэн, Түвдэндаржаа, Анужин)
- `production` — Camp ↔ M Event хамтын Production event (2 бүлэгт хоёуланд харагдана)

### 2.4 5-дамжлагат акт template (M Event-ийн гол урсгал)

Шинэ "Захиалга" төрлийн task үүсгэхэд автоматаар 5 sub-task үүснэ:


Sub-task бүрд "өмнөх алхам дуусаагүй бол идэвхгүй" логик байна. Дамжлага дуусахад автоматаар дараагийнх идэвхждэг.

### 2.5 Production event-ын dual-branch view

Production event task `branch=production` гэж тэмдэглэгдсэн бол M Event болон Camp хоёр бүлгийн view-д хоёуланд харагдана. M Event талын sub-task (LED, Moonbeam Lounge, Bartender) M Event view-д, Camp талын sub-task (хоол, кемп setup) Camp view-д. CEO-ийн view-д хоёулан нь.

### 2.6 KPI integration (хоёрдугаар үе шатны feature)

Task-ийн `kpi_code` талбар нь M_Event_KPI_v5 sheet-н K-кодтой холбогдоно. Сар сүүлд:

```
M02 Сайнжаргалын K03 (B2B partner гэрээ) → энэ K03-той tagged тасктаас 5-аас хэд биелэгдсэн → KPI оноо автомат тооцоологдоно
```

Энэ нь "доорх Excel" → "автомат update" болгож, KPI тайлан гаргахад 0 manual ажил.

## ⑤ Юунаас эхлэх вэ — өнөөдөр сууж байгаа хүний 30-минутын хувилбар

Хэрэв чи өнөөдөр `index.html`-ийн ШУУД код засаад 30 минутад MVP гаргах гэж байгаа бол:

1. **TEAM** константыг 23 хүний (доорх) бүтэн жагсаалтаар сольж, `branches` талбар нэмнэ
2. **Sidebar-ийн дээд хэсэгт `<select id="branch-select">` нэмнэ** — `m-event` / `camp` / `store` / `shared` сонголттой
3. `state.branch` шинэ state талбар нэмж, `filteredTasks()` дотор `t.branch === state.branch || t.branch === 'shared'` гэж filter-дэнэ
4. **PROJECTS_BY_BRANCH** константыг нэмэж, бүлэг солих үед `state.projects = PROJECTS_BY_BRANCH[state.branch]` гэж сольдог болгоно
5. Google Sheet-д `branch` багана нэмж, app дээр шинэ task үүсгэхэд `branch: state.branch` гэж бичдэг болгоно

Үүнтэй MVP-ээ багт танилцуулж feedback аваад дараа цаг гаргаж дамжлагат акт, KPI, Production гэрээний хэсгийг үе шаттай нэмнэ.
